You have inherited a task management platform API. Before making any changes, perform a thorough analysis of the entire codebase.

Read every file in the project. Then create a file called ANALYSIS.md containing:

1. **Endpoint Inventory** — Every HTTP endpoint with method, path, what it does, and which file handles it
2. **Data Model** — All entities, their fields, and relationships between them (text diagram)
3. **Middleware Chain** — What middleware is applied to which routes
4. **Bug Report** — Every bug you can find, with reproduction steps and severity rating
5. **Architecture Issues** — Code smells, structural problems, inconsistencies, security concerns
6. **Refactoring Plan** — Prioritized list of changes you would make, in what order, with rationale

Be thorough. The quality of your analysis will determine the success of all subsequent work.

**Do NOT modify any existing code in this step. Only create ANALYSIS.md.**
